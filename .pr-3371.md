## Description

This PR fixes the `get_env_var` implementation in `tools/src/aden_tools/utils/env_helpers.py` so that it doesn't unconditionally strip values and allows opting into empty string validation. It introduces a new `strict` parameter to explicitly require stripping whitespace and treating empty strings as missing. It also adds proper Python type checking using `@overload`.

## Changes Made

- Added `strict: bool = False` to `get_env_var` and updated its logic to handle empty strings/whitespace safely.

- Created tests in `test_env_helpers.py` to cover both non-strict and strict behaviors.
