## Description

Implemented the Circuit Breaker pattern for MCP tool calls to track server degradation and prevent cascading delays when a server starts failing.

## Changes Made

- Created `CircuitBreaker`, `CircuitStats`, `CircuitState`, and `CircuitBreakerRegistry` in `core/framework/runner/circuit_breaker.py`.
- Integrated `CircuitBreaker` into `MCPClient.call_tool()` to accurately fail fast on network/server errors and recover gracefully.
- Exposed `get_server_health` and `get_health_report` in `ToolRegistry` to query health statuses across servers.
- Added comprehensive unit tests in `core/tests/test_circuit_breaker.py` mapping states and verifying thread safety.
