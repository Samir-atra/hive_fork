## Description

Resolves #2967

Adds user-facing ROI business value tracking by computing the exact cost of each LLM execution call. The cost propagates up from the lowest stream chunks (via `litellm.cost_calculator.completion_cost()`) up to the final RunSummary.

## Changes Made

- Add `cost` property to `LLMResponse`, `FinishEvent`, `NodeResult`, `Outcome`, and `RunSummary`.
- Add `total_cost` property to `ExecutionResult`, `RunMetrics`, and `SessionProgress`.
- Extract token costs directly via LiteLLM (`litellm.cost_calculator.completion_cost()`) for standard completions and chunk-based event streams.
- Accumulate and propagate the calculated token cost inside `GraphExecutor.execute` (including parallel branches) and `EventLoopNode.execute`.
- Add unit tests verifying cost accumulation in `core/tests/test_cost_metrics.py`.
