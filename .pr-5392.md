## Description

This PR implements a multi-layered resilience strategy to protect Hive from data integrity risks and cascading failures during service interruptions.
It addresses the issue of data corruption mid-write, brittle recovery from malformed session files, and redundant request floods on prolonged outages.

## Changes Made

- Created `core/framework/utils/resilience.py` with `CircuitBreaker` utility class.
- Modified `FileConversationStore._write_json` to use `atomic_write` temp-file-rename pattern to prevent data corruption.
- Modified `FileConversationStore._read_json` and `SessionStore.read_state`/`list_sessions` to isolate corrupted JSON state files to `.corrupted/` directory.
- Integrated `CircuitBreaker` into `ExecutionStream` to fail fast and reject requests when entry points encounter repeated errors.
- Added comprehensive tests for circuit breaker, atomic writes, and corruption isolation in `core/tests/test_resilience.py`.
