## Description

This PR fixes a bug where the backend would fail to create a session on Windows due to an `Invalid format string` error. The issue was caused by the use of the `%-d` POSIX-only modifier in `strftime` when formatting episodic memory dates. This modifier is not supported by the Windows C runtime and throws a `ValueError`.

To resolve this, the PR replaces occurrences of `date.today().strftime("%B %-d, %Y")` with a cross-platform alternative using standard f-string formatting (`f"{today.strftime('%B')} {today.day}, {today.year}"`), ensuring that date formatting is consistent and safe across all operating systems.
