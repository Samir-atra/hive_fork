## Description

This PR introduces a business-friendly intervention interface by providing a comprehensive `Intervention` schema with an integrated audit trail, along with backend API endpoints for seamless approval workflows.

## Changes Made

- Created `Intervention`, `AuditLog`, and `InterventionStatus` schemas in `core/framework/schemas/intervention.py`.
- Added RESTful endpoints (`/api/interventions`, `/api/interventions/{id}/approve`, etc.) in `core/framework/server/routes_interventions.py` to list, create, approve, reject, and escalate interventions.
- Registered the new intervention routes in `core/framework/server/app.py`.
- Added tests for the new schemas and API routes in `core/tests/test_server_interventions.py`.
- Updated API documentation in `core/framework/server/README.md`.
