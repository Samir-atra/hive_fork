## Description

This PR introduces an opt-in storage retention and compaction policy for file-based runtime storage (`RuntimeLogStore`). Currently, local file-based logs grow without bounds which is an issue for long-running agents. This PR addresses this by allowing the configuration of retention limits based on maximum runs, log age, and disk usage.

## Changes Made

- Added `StorageRetentionPolicy` model to `runtime_log_schemas.py` supporting `max_runs`, `max_age_days`, and `max_disk_mb` configurations.
- Hooked up `StorageRetentionPolicy` into `RuntimeConfig` and `AgentRuntimeConfig` for configurability via `~/.hive/configuration.json` or explicitly in Python.
- Implemented `prune` async method in `RuntimeLogStore` to evaluate constraints and delete old runs via `shutil.rmtree`.
- Passed the storage retention policy from `ExecutionStream` to `RuntimeLogger` so the pruning triggers automatically inside `RuntimeLogger.end_run()`.
- Created tests to verify proper enforcement of pruning conditions (`max_runs`, `max_age_days`, `max_disk_mb`).
- Updated `RUNTIME_LOGGING.md` to document the new configurations.
