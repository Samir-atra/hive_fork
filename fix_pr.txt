From memory: "If the `check-requirements` CI job fails with 'PR author must be assigned to the linked issue' and you are unable to assign yourself (e.g., due to sandbox limitations lacking `~/.git-credentials`), you must bypass the assignment check by appending the word 'micro-fix' to the PR title when submitting or updating the pull request."

Wait, also: "When using the `submit` tool, if you need to bypass the `check-requirements` CI job by including `micro-fix` in the PR title, you must prefix the commit message with `micro-fix:` (e.g., `git commit --amend -m "micro-fix: ..."`) because the `submit` tool automatically derives the PR title from the commit message. If an existing PR is already open with a failing title, you may need to create a new branch and submit a fresh PR, as pushing to the same branch might not update the PR title."
