## Description

Fixes the token estimation logic in `NodeConversation` to accurately estimate tokens for JSON/code and CJK text. The previous heuristic (`total_chars // 4`) significantly underestimated the token count for structured data, leading to context overflow errors in code-heavy agents. This PR introduces an adaptive token estimation heuristic and lowers the default compaction threshold to 70% to provide a better safety margin.

## Changes Made

- Updated `NodeConversation.estimate_tokens` to use an adaptive heuristic for different content types (~2.5 chars/token for JSON/code, ~1 char/token for CJK, ~4 chars/token for english).
- Lowered the default `compaction_threshold` in `NodeConversation` from `0.8` to `0.7`.
- Added unit tests to `test_node_conversation.py` to verify the new adaptive token estimation heuristics and the new compaction threshold default.
