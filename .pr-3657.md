## Description

Introduce a unified, first-class `Failure` model that can be used across runtime execution, tool invocation, and evaluations. This model wraps existing error concepts rather than wholesale replacing them, using optional structured objects.

## Changes Made

- Created `Failure` pydantic model and `FailureType` enum in `core/framework/schemas/failure.py`.
- Updated `core/framework/schemas/__init__.py` to export the new models.
- Updated `core/framework/schemas/decision.py` to optionally include `Failure` object in `Outcome`.
- Updated `core/framework/schemas/run.py` to optionally include `Failure` object in `Problem` when creating new instances.
- Added comprehensive unit tests for instantiation and serialization logic in `core/tests/framework/schemas/test_failure.py`.
