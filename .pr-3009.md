## Description

This PR introduces the ability to add custom tools declaratively via YAML or JSON definitions, eliminating the need to write Python code for simple tools. The tools are defined within an agent's `tools/` directory (or subdirectories) and are automatically loaded during the agent's initialization. This allows execution mapping for Python functions, shell commands, or HTTP endpoints.

## Changes Made

- Implemented `discover_from_declarative_dir` in `ToolRegistry` to parse `.yaml` and `.json` declarative tool definitions.
- Created executor factories to securely map tools to Python functions, shell commands, and HTTP API endpoints.
- Updated `AgentRunner._setup()` to scan the agent's `tools/` directory and any of its immediate subdirectories.
- Added comprehensive unit testing to cover parsing, execution, and environment variable/URL templating mappings in the new `ToolRegistry` logic.
