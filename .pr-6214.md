## Description

This PR activates the `NodeSpec.model` field for per-node model selection and adds a `DegradationPolicy` for automatic fallback when a node exceeds its token budget. It allows executors to seamlessly transition to a cheaper fallback model automatically once the defined token budget is hit for a specific node execution.

## Changes Made

- Added `DegradationPolicy` and `degradation_policy` field to `NodeSpec` (in `core/framework/graph/node.py`).
- Added abstract `with_model` method to `LLMProvider` (in `core/framework/llm/provider.py`) and implemented it in `LiteLLMProvider`, `AnthropicProvider`, `AntigravityProvider`, and `MockLLMProvider`.
- Modified `GraphExecutor._build_context` to support per-node `NodeSpec.model` overriding and trigger `DegradationPolicy.fallback_model` when tokens usage across iterations surpasses `token_budget`.
- Monitored `node_token_counts` across visits in `GraphExecutor`.
- Added unit tests for degradation policy in `core/tests/test_degradation_policy.py`.
