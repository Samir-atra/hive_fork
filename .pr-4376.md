## Description

This PR introduces a failure-aware agent behaviour pattern that allows agents to explicitly surface missing context or prerequisites before executing actions. It introduces a new `report_missing_context` tool that allows agents to state what is missing, why it is required, and how to resolve the gap, before pausing execution to wait for user or supervisor guidance.

## Changes Made

- Introduced a new synthetic tool `report_missing_context` in `EventLoopNode`.
- Handled the `report_missing_context` tool by emitting a new event, logging the gap, and pausing execution.
- Added `MISSING_CONTEXT_REPORTED` to `EventType` and defined `emit_missing_context_reported` in `EventBus`.
- Updated `EVENT_TYPES.md` with documentation for the new event.
- Added tests to verify the tool successfully blocks execution and emits the proper event.
