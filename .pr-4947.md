## Description

Fixes a bug in `ConcurrentStorage` where the in-memory cache could hold data that was never successfully written to disk (e.g., due to write failures). It tracks pending writes for batched queues to ensure that `load_run` can fetch runs immediately after queuing, and implements a retry mechanism for batched writes, eventually invalidating the cache if all retries fail.

## Changes Made

- Added `self._pending_writes` dictionary to `ConcurrentStorage` to track runs that are queued but not yet written.
- Modified `save_run` to evict the cache on `immediate=True` write failures, and track the run in `_pending_writes` for batched writes.
- Modified `load_run` to check `_pending_writes` before looking in the main cache, allowing callers to fetch queued runs correctly.
- Modified `_flush_batch` to retry failed run saves up to 3 times with a 0.5s delay.
- Modified `_flush_batch` to explicitly remove the run from cache if all save retries fail, and from `_pending_writes` regardless of success.
- Modified `delete_run` to ensure `_pending_writes` is correctly updated.
- Added tests in `TestConcurrentStorageEdgeCases` to verify correct behavior.
