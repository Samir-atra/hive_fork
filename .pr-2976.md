## Description

This PR fixes the issue where the CLI agent execution output was silently filtering un-"meaningful" output keys, causing confusion for agent authors and users. It adds an optional `--full` flag to the `run` and `shell` CLI commands, which displays the entire unmodified agent output in JSON when supplied. Additionally, when output keys are filtered out due to the `--full` flag not being set, a helpful notice `(Output has been filtered. Use --full to see all output keys)` or `(Output has been filtered. Use /full to see all output keys)` is printed to inform the user of this behavior. A `/full` toggle command has also been added to the interactive agent shell loop to easily toggle this behavior dynamically.

## Changes Made

- Added `--full` argument to the CLI `run_parser` and `shell_parser` configurations in `core/framework/runner/cli.py`.
- Updated `cmd_run` output printing logic to print the full JSON dumped `result.output` string when `--full` is provided, and else fallback to the original default behavior while outputting a helpful note about the behavior.
- Updated `cmd_shell` similarly by tracking a local `full_output` state initialized by `--full`.
- Implemented `/full` interactive session command logic in `cmd_shell` to toggle the local output filter state on/off.
