## Description

This PR implements Phase 1 of the Production Security Hardening and Enterprise Backend Integration Layer by adding robust, configurable security middlewares to the `aiohttp` API server.

## Changes Made

- Added `ServerConfig` parsing to `core/framework/config.py` to allow dynamic `api_key`, `rate_limit_requests`, `rate_limit_window`, and `cors_origins`.
- Implemented `api_key_auth_middleware` with safe string comparison via `hmac.compare_digest`.
- Implemented `rate_limit_middleware` with a basic garbage-collected memory store to track IP requests.
- Updated `cors_middleware` to use configured origins (supporting `*` and custom lists).
- Included full unit test coverage for middlewares in `test_middleware.py`.
