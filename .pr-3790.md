## Description

This PR introduces a static `GraphAnalyzer` for pre-execution validation of agent graphs. It is designed to catch structural issues—such as unreachable nodes, dangerous cycles, missing inputs, and dead-end paths—statically before runtime, saving tokens and debugging time.

## Changes Made

- Created `core/framework/graph/analyzer.py` implementing `GraphAnalyzer` with comprehensive BFS/DFS-based static analysis capabilities (dead-end paths, unreachable nodes, cycle detection handling `max_node_visits`, missing inputs, key collisions).
- Added `AnalysisReport` with error/warning/info severity levels and token cost estimation.
- Integrated the analyzer into the `hive validate` CLI command in `core/framework/runner/cli.py`.
- Added `--strict` (fail on warnings) and `--json` (machine-readable output) flags to the `hive validate` command.
- Created robust unit tests in `core/tests/test_graph_analyzer.py` exercising the new checks.
