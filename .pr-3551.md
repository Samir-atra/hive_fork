## Description

This PR fixes a bug where `GraphExecutor.register_node()` accepted any object without validation (including `None`, integers, strings, etc.), leading to cryptic errors during execution time. By validating the implementation up front, errors are caught at registration time.

## Changes Made

- Added a check for `None` in `register_node` raising `ValueError("Cannot register None as node")`.
- Added an `isinstance` check to ensure the passed implementation is of type `NodeProtocol`.
- Verified that the `execute` method exists on the implementation and is an async coroutine, raising a detailed `ValueError` if not.
- Added comprehensive unit tests targeting invalid registrations under `core/tests/framework/graph/test_invalid_node_types.py`.
