## Description

This PR introduces the "Agent Composability" feature by adding an `agent_invoke` node type. This allows an agent to act as a parent and call another agent (sub-agent) entirely, effectively treating it as a callable tool with its own isolated memory sandbox.

## Changes Made

- Added `agent_ref` and `input_mapping` properties to `NodeSpec` in `core/framework/graph/node.py` to support specifying the sub-agent ID/path and mapping variables from the parent memory to the sub-agent.
- Created `AgentInvokeNode` in `core/framework/graph/agent_invoke_node.py` which implements `NodeProtocol`. It loads the sub-agent via `load_agent_export`, maps memory values based on `input_mapping`, and recursively instantiates `GraphExecutor` to execute the sub-agent in a secure, isolated context.
- Registered `"agent_invoke"` as a valid node type in `VALID_NODE_TYPES` inside `GraphExecutor` (`core/framework/graph/executor.py`) and implemented the instantiation logic for `AgentInvokeNode`.
