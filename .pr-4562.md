## Description

This PR implements an SDK for defining failure conditions as part of the Roadmap Eval System, enabling more expressive, assertion-like testing workflows for agents.

## Changes Made

- Created `FailureCondition` base class in `core/framework/testing/failure_conditions.py`.
- Implemented specific subclasses: `StringMatchCondition`, `ErrorTypeCondition`, and `MetricRangeCondition`.
- Updated `Test` schema in `core/framework/testing/test_case.py` to allow attaching an array of `FailureCondition`s via a new `failure_conditions` field.
- Exported the new components in `core/framework/testing/__init__.py`.
- Added unit tests in `core/tests/test_failure_conditions.py` to verify functionality.
