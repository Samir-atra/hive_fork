## Description

This PR fixes an issue where the CLI (`hive run`, `hive dispatch`, and `hive shell`) would crash and dump raw Python tracebacks when `asyncio.run()` encounters exceptions (such as an LLM `AuthenticationError` from LiteLLM). It implements try/except blocks to gracefully catch and report errors.

## Changes Made

- Added `_format_error(e: Exception) -> str` to format common LLM errors (like `AuthenticationError`) into user-friendly strings without tracebacks.
- Wrapped `asyncio.run()` in `cmd_run` with `try...except KeyboardInterrupt` (exits 130) and `Exception` (prints formatted error and exits 1).
- Wrapped `asyncio.run()` in `cmd_dispatch` similarly.
- Wrapped `asyncio.run()` in the `cmd_shell` interactive loop (single agent) with `try...except`, using `continue` to preserve session memory on failure instead of exiting.
- Wrapped `asyncio.run()` in the `cmd_shell` interactive loop (`_interactive_multi` orchestrator) with `try...except`, preserving the session loop on failure.
