## Description

This PR implements the State Checkpointing Layer as proposed in #3663, enabling an agent to resume from the last successful node instead of restarting from the entry point upon failure or a pause event.

## Changes Made

- Added `last_successful_node_id` to `SessionProgress` and `ExecutionResult`.
- Modified `GraphExecutor.execute` to track and update the `last_successful_node_id` on node success.
- Modified `GraphExecutor._write_progress` to persist the watermark to `state.json`.
- Updated `GraphExecutor.execute` entry point fallback logic to resume from `last_successful_node_id` when `paused_at` is missing or invalid.
- Updated `ExecutionStream` to restore `last_successful_node_id` from the session state upon recovery.
- Updated `cli.py` to restore the watermark in the resumed state payload.
