## Description

This PR introduces AWS Secrets Manager and Azure Key Vault as new credential storage backends, bringing them up to parity with the existing `HashiCorpVaultStorage`. This allows enterprise teams to securely retrieve API keys, database credentials, and other secrets directly from their cloud provider's native secrets management services, without forcing users to sync secrets to environment variables manually.

## Changes Made

- Created `AWSSecretsManagerStorage` using `boto3` for AWS Secrets Manager integration.
- Created `AzureKeyVaultStorage` using `azure-identity` and REST endpoints for Azure Key Vault integration.
- Added comprehensive unit tests in `core/tests/credentials/` for both storage backends with fully mocked dependencies.
- Added optional `aws` and `azure` dependency groups in `core/pyproject.toml`.
- Updated `docs/credential-store-design.md` and `docs/credential-store-usage.md` to document the newly implemented storage options and their required setup.
