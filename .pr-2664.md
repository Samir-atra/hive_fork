## Description

Resolves #2664 by making the `Runtime` class thread-safe.

The `Runtime` class maintained shared execution state across threads via `_current_run` and `_current_node`. In an async or multi-threaded context, sharing a single `Runtime` instance interleaves context and leads to execution state corruption.

This patch refactors `Runtime` to use a `threading.local()` object, properly scoping internal execution variables per thread.

## Changes Made

- Replaced shared `self._current_run` and `self._current_node` instance attributes in `Runtime.__init__` with `self._local = threading.local()`.
- Updated all getters, setters, and class methods to retrieve and modify execution context using `getattr(self._local, "current_run", None)` and `getattr(self._local, "current_node", "unknown")`.
- Authored a comprehensive thread isolation pytest module (`test_runtime_thread_safety.py`).
- Updated the `Runtime` docstring to explicitly declare its thread-safe nature.
