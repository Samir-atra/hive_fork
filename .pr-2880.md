## Description

This PR adds a 30-second timeout to the `_llm_route` method in `AgentOrchestrator`. Previously, if the LLM provider became slow or unresponsive, the orchestrator could hang indefinitely because `self._llm.acomplete()` was called without a timeout. This change wraps the LLM call with `asyncio.wait_for` to ensure execution fails fast or falls back gracefully, significantly improving system reliability.

## Changes Made

- Wrapped `self._llm.acomplete()` inside `_llm_route()` in `core/framework/runner/orchestrator.py` with `asyncio.wait_for(..., timeout=30.0)`.
- Added a unit test `test_llm_route_timeout` in `core/tests/test_orchestrator.py` to ensure the orchestrator correctly times out and falls back to the highest confidence agent instead of hanging.
