## Description

Fixes the misleading UI state in the CredentialsModal where it shows "No required credentials!" but then persists a "Missing credentials — configure them to continue" warning banner in the workspace on Windows. The core issue was that the `POST /api/sessions` call encountered a `CredentialError` during Queen initialization but without a `validation_result` attached, leading to an empty `required` array in the 424 response payload.

## Changes Made

- **Backend (`core/framework/server/session_manager.py`)**: Synthesized a `CredentialValidationResult` and attached it to the `CredentialError` during `LiteLLMProvider` or `AntigravityProvider` initialization failure.
- **Frontend (`core/frontend/src/pages/workspace.tsx`)**: Captured the `errBody.required` from the 424 response and passed it down to `CredentialsModal` as `directRequirements`.
- **Frontend (`core/frontend/src/components/CredentialsModal.tsx`)**: Utilized `directRequirements` inside `fetchStatus` to display the missing keys correctly, and updated the "Done" button `onClick` handler to call `onCredentialChange?.()` when all required credentials are met, ensuring the warning banner is cleared upon modal closure.
