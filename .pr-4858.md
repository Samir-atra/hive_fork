## Description

This PR resolves multiple bug reports regarding `pdf_read_tool.py`, specifically file descriptor resource leaks, edge cases in parsing requested page ranges, non-standard PDF date strings formatting, and lacking feedback for image-only/scanned PDFs.

## Related Issues

Resolves #4858

## Changes Made

- Replaced open `PdfReader(path)` handling with a `with open(path, "rb") as f:` context manager block to fix resource leaks and unclosed files warning.
- Enhanced `parse_page_range` to strip whitespace, support multi-format inputs (e.g. `1, 2, 5-10`), correctly handle index order, and allow negative indices (`-1` corresponds to the last page).
- Added a fallback ISO-8601 formatting helper, `_parse_pdf_date(date_str)`, to convert non-standard PDF date strings such as `D:YYYYMMDDHHmmSSZ` properly for `/CreationDate` and `/ModDate` metadata tags.
- Verified final extracted string size, adding `result["extracted_text_empty"] = True` alongside an explicit warning to prompt users for OCR processing if text cannot be found natively within the PDF.
