## Description

This PR fixes a bug where `ParallelExecutionConfig.branch_timeout_seconds` was not properly encapsulated in a wrapper function during parallel execution, potentially leading to inconsistencies. The logic has been restructured to cleanly wrap the `execute_single_branch_core` function with the `asyncio.wait_for` timeout logic, preserving the core execution behavior (including retries) while ensuring timeouts are caught and converted to the proper `branch.status = "timed_out"` state.

## Changes Made

- Renamed the inner branch execution function to `execute_single_branch_core`.
- Added a new `execute_single_branch` wrapper function that executes the core logic within an `asyncio.wait_for` timeout.
- Handled the resulting `TimeoutError` from inside the wrapper instead of the main collection loop, setting `branch.status = "timed_out"` immediately and returning the branch alongside the exception.
- Cleaned up the redundant `asyncio.TimeoutError` exception matching in the outer loop result processing since the wrapper directly emits the encapsulated exception.
