## Description

This PR fixes timezone-naive datetime fields across the codebase by adopting timezone-aware datetimes with the `UTC` timezone. Previously, `datetime.now()` and `datetime.utcnow()` were creating naive objects, which lead to serialization warnings and timezone ambiguity in distributed scenarios.

## Changes Made

- Updated Pydantic field default factories to use `lambda: datetime.now(UTC)` instead of `datetime.now`.
- Replaced explicit `datetime.now()` calls with `datetime.now(UTC)` in schemas and framework files.
- Replaced `datetime.utcnow()` with `datetime.now(UTC)` in `CredentialLocal` models.
- Updated related tests to assert against timezone-aware datetimes.
