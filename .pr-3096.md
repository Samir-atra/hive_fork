## Description

This PR introduces a comprehensive two-part integration to make the Hive framework more stateful and ready for real-time observability in production environments, fully addressing #3096.

First, it integrates the newly added `NodeConversation` directly into `NodeContext`, ensuring persistent, disk-backed (via `FileConversationStore`) conversation histories across execution nodes.

Second, it implements a pub-sub Real-Time Streaming Interface (`ExecutionStream` and `ExecutionEvent`) connected deeply into the `GraphExecutor`'s lifecycle. We now emit critical runtime events including `run.started`, `node.started`, `message.added`, `node.completed`, `run.completed`, and `error`. Additionally, initial stub commands (`monitor` and `conversation`) are introduced in the CLI.

## Changes Made

- **NodeConversation Integration:** Updated `NodeContext` and `GraphExecutor` to instantiate and expose conversation memory dynamically, automatically loading via `FileConversationStore` when `enable_conversation` is true. Added `add_user_message` and `add_assistant_message` to the context.
- **Real-Time Streaming Interface:** Created `ExecutionEvent` and `ExecutionStream` in `core/framework/streaming`.
- **Event Lifecycle Hooks:** Wired streaming logic directly into `GraphExecutor`, firing terminal lifecycle events correctly through asynchronous shielded tasks with strong references to prevent premature task GC. Added `message.added` emissions when writing to the conversation object.
- **CLI Addition:** Added new `monitor` and `conversation` commands to the runner. `conversation` allows inspecting and exporting node messages.
