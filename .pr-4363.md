## Description

Add a vendor-agnostic OpenTelemetry adapter layer that takes Hive's existing internal telemetry data (from EventBus, OutcomeAggregator) and exports it to standard monitoring tools. This allows pushing logs and traces seamlessly to Datadog, Grafana, Jaeger, etc. without duplicating the existing event logic.

## Changes Made

- Created `OTELExporter` in `core/framework/observability/otel_adapter.py` to listen to EventBus lifecycle events and create W3C-compliant traces.
- Created `MetricsAdapter` in `core/framework/observability/otel_adapter.py` to poll OutcomeAggregator for goal progress and push metric gauges.
- Exported the newly created classes to `core/framework/observability/__init__.py`.
- Added OpenTelemetry API, SDK, OTLP exporter, and Prometheus exporter to `project.optional-dependencies` under the new `observability` extra in `pyproject.toml`.
