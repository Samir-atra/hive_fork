## Description

This PR improves conversational behaviors in the `EventLoopNode`. It prevents client-facing nodes with no output keys from being pressured by the implicit judge, injects a synthetic `end_conversation` tool allowing these nodes to terminate gracefully, and introduces a warning at 80% of `max_iterations` to allow the conversational model to wrap up nicely before exiting successfully at the limit.

## Changes Made

- Edited `_judge_turn` to return empty feedback instead of injecting pressure when `output_keys` is empty and `client_facing` is True.
- Added `end_conversation` synthetic tool alongside other tools.
- Modified the main loop to inject a system warning message at 80% `max_iterations`.
- Changed the max iteration exit condition from a hard failure to a successful termination for client-facing nodes.
