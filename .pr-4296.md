## Description

Resolves #4296. This PR addresses the unsafe lazy initialization pattern in `AgentRunner._setup()`. Due to `_setup()` being called lazily from multiple asynchronous coroutines, concurrent invocations could result in the method being executed multiple times, duplicating resource allocation. This fix uses a lazy `asyncio.Lock` mechanism (`_ensure_setup`) to ensure that `_setup()` acts as a singleton call in async environments, effectively preventing the race condition.

## Changes Made

- Added `_setup_lock` to `AgentRunner` to prevent concurrent lazy setup execution
- Implemented `_ensure_setup()` as an async handler that securely utilizes the lock
- Refactored multiple async entry methods in `AgentRunner` (`can_handle`, `get_goal_progress`, `trigger`, `start`, `_run_with_agent_runtime`, `__aenter__`) to use `await self._ensure_setup()` instead of invoking `self._setup()` directly.
- Added regression test `test_runner_setup_lock.py` to ensure thread-safety across multiple `start()` invocations.
