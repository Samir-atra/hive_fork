## Description

Resolves #2531

Fixes an issue where `EventBus.publish` would deadlock the system when concurrent handlers saturated the bounded semaphore. By dispatching handlers as fire-and-forget tasks (`asyncio.create_task`), `publish` now returns immediately and won't block the caller (e.g. an agent execution loop). For testing synchronization, a new `_wait_for_handlers` internal flag and a `wait_until_idle()` method have been added and appropriately implemented in the test suite.

## Changes Made

- Modified `EventBus.publish` to execute handlers asynchronously as fire-and-forget background tasks.
- Introduced `_wait_for_handlers` flag to `EventBus.publish` to optionally block until handlers complete, primarily used for tests.
- Introduced `wait_until_idle` to `EventBus` to await all running background handler tasks (handles chained executions with a `while` loop).
- Updated existing tests in `test_event_loop_integration.py`, `test_event_loop_node.py`, `test_event_type_extension.py`, and `test_session_manager_worker_handoff.py` to synchronously await side effects before asserting logic.
- Added `test_event_bus_publish_fire_and_forget_deadlock_prevention` in `test_event_bus_deadlock.py` to verify the bug fix.
