## Description

This PR fixes an issue where user input context sent through the main chat reaches only the queen agent and is not propagated to active worker runs that are awaiting input. By dual-routing chat messages, workers can now act on new user context seamlessly.

## Changes Made

- Modified `handle_chat` in `core/framework/server/routes_execution.py` to check for an awaiting worker node using `find_awaiting_node()`.
- Added logic to inject the chat message to the worker runtime alongside the existing queue to the queen agent.
- Updated the response JSON to correctly return the `"injected"` status and the awaiting `node_id` when the message is successfully propagated to the worker, ensuring the frontend gracefully acknowledges the delivery.
