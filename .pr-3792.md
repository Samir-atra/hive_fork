## Description

Resolves #3792

Tool results in Hive flow from external MCP servers directly into the LLM conversation context with zero sanitization. A malicious or compromised tool could return content that hijacked the context. This PR introduces a `PromptInjectionShield` to scan and detect indirect prompt injection.

## Changes Made

- Add `PromptInjectionShield` to detect indirect prompt injections based on regex patterns (instruction override, role hijacking, information extraction, delimiter escape, command injection).
- Integrate the shield in `EventLoopNode`, scanning tool result text and wrapping it in `<tool_result trust="untrusted">` when malicious patterns are found, or blocking entirely if `block` mode is used.
- Propagate `prompt_injection_shield` config in `GraphSpec`, `LoopConfig` and execution pipeline in `executor.py` and `runner.py`.
- Verified logic with a comprehensive test suite.
