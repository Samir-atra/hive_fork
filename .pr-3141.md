## Description

This PR implements per-workflow cost tracking for LLM API calls, resolving issues #3141 and #506. It accurately attributes LLM costs generated from both standard completion calls and streaming events directly to the `workflow_id` level. A new `LLMCostCalculator` delegates logic to `litellm`, populating `LLMResponse`, `FinishEvent`, `NodeResult`, `ExecutionResult`, `Run`, and `RunMetrics` with precise cost numbers enabling robust ROI analysis.

## Changes Made

- Add `LLMCostCalculator` in `core/framework/llm/cost.py` using `litellm`
- Add `cost_usd` properties to `LLMResponse`, `NodeResult`, `FinishEvent`, and `Outcome` schemas
- Aggregate `total_cost_usd` and `cost_by_model` inside `ExecutionResult` and tracking loops inside `GraphExecutor`
- Aggregate and emit per-turn iteration cost tracking inside `EventLoopNode`
- Track `workflow_id` natively within the `Run` schema initialization in `AgentRuntime` and `StreamRuntime`
- Provide accurate and deterministic fallback matching ($0.0) when LLM pricing data isn't available
- Update all corresponding schema unit tests and include specific unit testing for `LLMCostCalculator` fallback logic
