## Description

This PR implements the Persistent Developer Feedback Loop (Roadmap) feature requested in #2731. It allows developers to register custom prompt feedback that gets persistently injected into agent contexts across runs without requiring code/template modifications.

## Changes Made

- Created `core/framework/adaptiveness/feedback_store.py` providing a lightweight persistent JSON-backed `FeedbackStore` singleton.
- Modified `core/framework/graph/executor.py` (`_build_context` method) to inject global developer instructions from the `FeedbackStore` into the narrative prompt layer prior to node execution.
- Added a `hive feedback <instruction>` CLI command to `core/framework/runner/cli.py` to allow developers to rapidly register these persistent instructions via terminal.
- Added comprehensive unit test coverage for the new feature at `core/tests/framework/adaptiveness/test_feedback_store.py`.
