## Description

Fixes an issue where the Pipeline UI incorrectly displays "complete" and shows green nodes when the core execution fails globally (e.g., due to missing tools).

## Changes Made

- Update `workspace.tsx` to handle `execution_failed` events when no `node_id` is present.
- Change `markAllNodesAs` logic in `execution_failed` to mark all nodes as `error` rather than reverting to `pending` when the execution fails globally (i.e., `!event.node_id`). This visually indicates that the pipeline as a whole encountered a failure preventing execution.
