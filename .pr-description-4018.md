## Summary

Adds comprehensive test coverage for `core/framework/graph/safe_eval.py` - a security-critical sandboxed expression evaluator used by `EdgeSpec.condition_expr`.

## Test Coverage

The new test suite includes 740+ lines covering:

### Core Functionality
- **Literals and constants**: integers, floats, strings, booleans, None
- **Data structures**: lists, tuples, dicts, nested structures
- **Arithmetic operators**: +, -, *, /, //, %, **
- **Bitwise operators**: <<, >>, |, ^, &
- **Comparison operators**: ==, !=, <, <=, >, >=, is, is not, in, not in
- **Chained comparisons**: `1 < 2 < 3`
- **Unary operators**: +, -, not, ~
- **Boolean operators**: and, or
- **Ternary expressions**: `'yes' if condition else 'no'`

### Variable and Attribute Access
- Variable lookup from context
- Subscript access (list/dict indexing)
- Attribute access with safe method calls

### Whitelisted Functions
- Type conversions: `int`, `float`, `str`, `bool`, `list`, `dict`, `tuple`, `set`
- Aggregations: `len`, `min`, `max`, `sum`, `all`, `any`
- Math: `abs`, `round`

### Safe Methods
- Dict methods: `get`, `keys`, `values`, `items`
- String methods: `lower`, `upper`, `strip`, `split`

### Security Boundaries
- Private attribute access blocked (e.g., `__class__`, `_private`)
- Lambdas blocked
- Comprehensions blocked (list, dict, set, generator)
- Import/exec/eval blocked
- Function/class definitions blocked
- Assignment statements blocked
- Non-whitelisted functions blocked
- Non-whitelisted methods blocked

### Real-World Patterns
- Guard patterns: `x is not None and x.get('key')`
- Default value patterns: `output.get('key') or 'default'`
- Status checks: `status == 'success'`
- Type checks: `type(x) == list`
- Complex conditions: `output.success and output.score > 0.8`
