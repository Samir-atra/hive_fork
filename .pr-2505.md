## Description

This PR implements the Cost Feedback feature for developers running agents locally. When a developer executes an agent via the CLI (`hive run` or `hive shell`), the output will now include a real-time, detailed cost breakdown detailing total cost, costs by individual steps, and costs by the specific models used.

## Changes Made

- Added `cost` field to `LLMResponse` and `FinishEvent` data classes.
- Updated `litellm.py`'s `acomplete()`, `complete()`, and `stream()` methods to calculate execution cost using `litellm.completion_cost()`.
- Added `cost` and `models_used` fields to `NodeResult` and accumulated these throughout the `EventLoopNode`.
- Updated `RunMetrics` and `Outcome` models to hold `api_cost_usd`.
- Modified `ExecutionResult` to track `total_cost`, `cost_breakdown` by step, and `models_used`.
- Updated `GraphExecutor.execute` (including parallel fanout branch execution) to accurately aggregate metrics across the graph path.
- Formatted `cmd_run` and `cmd_shell` outputs in `cli.py` to match the required feature representation.
