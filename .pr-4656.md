## Description

Fix failing tests in `test_file_system_toolkits.py` on Windows due to Unix-specific shell commands (`ls`, `tr`, etc.). This change introduces platform conditional logic to use Windows equivalents when tests are run on Windows environments.

## Changes Made

- Updated `test_execute_command_list_files` to use `dir` on Windows and `ls` on Unix.
- Updated `test_execute_command_with_pipe` to use `powershell` for piping and uppercasing on Windows instead of `tr`.
- Normalized stdout assertions to handle `\r\n` Windows line breaks.
