## Description

Fixes a bug in `core/framework/credentials/__init__.py` where optional dependencies (like Aden sync components and Local credential registry) were exported in `__all__` regardless of whether they were successfully imported. If the optional dependencies were missing (e.g. `httpx`), importing `core.framework.credentials` with a wildcard (`from core.framework.credentials import *`) would crash with an `AttributeError`. The `__all__` list is now dynamically appended to only if the dependencies are successfully imported.

## Changes Made

- Removed hardcoded optional exports from the `__all__` list in `core/framework/credentials/__init__.py`.
- Updated the `try/except ImportError` blocks to append optional exports to `__all__` only on successful import.
- Added a new unit test in `core/tests/test_credentials_exports.py` that verifies the wildcard import works and optional exports are safely excluded when imports fail.
