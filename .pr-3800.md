## Description

This PR implements the `ShadowExecutor` to safely test graph evolution in shadow mode. It allows two versions of a graph (`baseline` and `candidate`) to run concurrently on identical input within isolated states. It then uses an LLM-based `VersionComparator` to evaluate both outcomes against the provided `Goal` and outputs a `ShadowComparisonResult`.

## Changes Made

- Created `ShadowExecutor` for parallel execution of isolated `GraphExecutor`s.
- Created `VersionComparator` which evaluates success, quality, token usage, and latency metrics to determine a winner between baseline and candidate graph versions.
- Added `ShadowComparisonResult` to capture the structured comparison metrics.
- Added `SHADOW_COMPARISON_COMPLETED` to `EventType` and `EventBus` to emit observability events for shadow comparisons.
- Implemented corresponding async unit tests in `core/tests/test_shadow_executor.py`.
