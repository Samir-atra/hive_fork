## Description

Fixes the `web_scrape` tool so that `max_length` correctly truncates text by byte count instead of character count. This ensures that multi-byte characters (such as emojis or non-ASCII characters) do not cause the final payload to silently exceed the intended byte size limit. It also ensures that the `...` truncation suffix is included within the `max_length` boundary, not appended after.

## Changes Made

- Truncate text logic updated to truncate based on `len(text.encode("utf-8"))`.
- `...` is appended inside the truncated boundaries.
- Re-decode is performed safely using `errors="ignore"` to avoid throwing an error if the truncation cuts off mid-character.
- Add byte-based max length limits test with emojis to `tools/tests/tools/test_web_scrape_tool.py`.
