## Description

This PR implements the missing goal constraint and success criteria evaluation logic in `GraphExecutor`. Prior to this change, goal constraints and success criteria were decorative. Now, constraints are checked after each node execution, preventing hard constraint violations by immediately terminating the run. Success criteria are validated at the end of execution to determine if the overarching goal was successfully met.

## Changes Made

- Updated `ExecutionResult` to include `constraint_violations`, `success_criteria_met`, and `goal_achieved`.
- Added `_evaluate_constraint` and `_evaluate_criterion` methods to `GraphExecutor` using `safe_eval`.
- Modified `GraphExecutor.execute` to evaluate constraints after every node is executed.
- Modified `GraphExecutor.execute` to immediately fail and return an `ExecutionResult` on hard constraint violations.
- Modified `GraphExecutor.execute` to evaluate success criteria at the end of the run.
- Added new test file `core/tests/test_goal_evaluation.py` to cover these features.
