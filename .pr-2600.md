## Description

This PR resolves #2600 by implementing a native artifact export mechanism inside the Hive runtime. At the end of every execution run, a deterministic artifact bundle is saved to the session storage (`runs/<run_id>/`). This allows external reviewers and operators to examine and replay exactly what nodes ran, how self-healing performed, and view human-in-the-loop (HITL) pauses via a visual map and event timeline.

## Changes Made

- Added `_export_run_artifact` method natively to `ExecutionStream` in `core/framework/runtime/execution_stream.py`.
- Configured the runtime to automatically write a bundle containing `graph.json`, `events.jsonl`, `graph.mermaid`, and `summary.md` on every run.
- Handled edge cases by wrapping file generation in fail-safe blocks to avoid interrupting active processes.
- Created `core/tests/test_run_artifact_export.py` to ensure the runtime creates the artifact directories and files properly.
