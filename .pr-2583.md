## Description

This PR fixes a concurrency violation in `SharedStateManager` where read operations bypassed locks when `IsolationLevel.SYNCHRONIZED` was used. The fix ensures that the `read` method correctly acquires the appropriate stream or global lock before accessing the shared state dictionaries.

## Changes Made

- Modified `core/framework/runtime/shared_state.py` to ensure `read` acquires locks when reading from stream or global state under `IsolationLevel.SYNCHRONIZED`.
- Added unit tests in `core/tests/test_shared_state.py` to verify that concurrent reads block correctly when a writer holds the lock.
