## Description

Introduces configurable execution safeguards (timeouts, retries, and exponential backoff) for tool calls within `EventLoopNode`.

## Related Issues

Resolves #5413

## Changes Made

- Added `tool_call_max_retries`, `tool_call_retry_backoff_base`, and `tool_call_retry_max_delay` to `LoopConfig`.
- Refactored `EventLoopNode._execute_tool` to wrap the tool execution inside an asynchronous retry loop.
- Automatically retries on `TimeoutError` and transient errors using `EventLoopNode._is_transient_error`.
- Retries utilize an exponential backoff formula with max delay capping.
- Added comprehensive unit tests in `test_tool_execution_retry.py`.
