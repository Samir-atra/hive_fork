## Description

This PR refactors the large `GraphExecutor.execute` method in `core/framework/graph/executor.py` into smaller, single-responsibility methods to improve readability, testability, and extensibility. Specifically, `_execute_node_implementation`, `_handle_retry`, `_handle_failure`, and `_handle_success` methods have been extracted from the main execution loop.

## Changes Made

- Extracted `_execute_node_implementation` to encapsulate obtaining node implementations, checkpoint generation, event bus emissions, node executions, GCU browser cleanups, runtime logging, and output validation.
- Extracted `_handle_retry` to encapsulate backoff logic and retry loop event emissions.
- Extracted `_handle_failure` to handle logic falling through the retry exhaustions via triggering failure routing or generating failing `ExecutionResult`.
- Extracted `_handle_success` to accumulate memory traces and log successful executions out of the main execution loop.
- Refactored the `GraphExecutor.execute` loop block body to correctly and seamlessly use these smaller single-responsibility methods without impacting current tracking mechanisms or state.
