## Description

Fixes a security issue in `grep_search` where it could traverse symlinks when executing recursive or non-recursive searches, potentially allowing access to files outside of the session sandbox.

## Related Issues

Resolves #2111

## Changes Made

- Edited `tools/src/aden_tools/tools/file_system_toolkits/grep_search/grep_search.py` to explicitly reject requests if `os.path.islink(secure_path)` evaluates to `True`, effectively closing a sandbox escape vulnerability via symlinked root search directory.
- Edited `os.walk` in `grep_search.py` to use `followlinks=False` to ensure that directory symlinks are not traversed in recursive search.
- Added explicit checks using `os.path.islink` to ensure file symlinks are skipped in both recursive and non-recursive scenarios.
- Updated `tools/tests/tools/test_file_system_toolkits.py` with four test cases checking symlink file searches and symlink directory traverse for both recursive and non-recursive search options.
- Edited `tools/src/aden_tools/tools/file_system_toolkits/grep_search/README.md` to note that symlinks are not supported.
